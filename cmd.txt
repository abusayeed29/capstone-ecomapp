Summary of Issues Resolved
Here are the key issues we encountered and fixed in your e-commerce Laravel app:

1. Database Seeding Failure: Duplicate Review Entries
Error: SQLSTATE[23000]: Integrity constraint violation: 1062 Duplicate entry '79-10' for key 'reviews_product_id_customer_id_unique'
Cause: Seeders tried to insert multiple reviews for the same (product_id, customer_id) pair, violating the unique constraint.
Fix: Updated seeders to use updateOrCreate for idempotency and added checks to skip duplicate reviews.
2. Admin Routes Not Visible in CLI
Error: php artisan route:list --path=admin showed no routes.
Cause: Filament Shield (permission system) wasn't set up, so admin routes weren't registered or accessible.
Fix: Ran php artisan shield:setup and php artisan shield:super-admin to initialize permissions and assign super admin role.
3. 403 Forbidden on Admin Panel Pages
Error: Accessing http://127.0.0.1:8000/admin/brands returned 403 Forbidden.
Cause: Permissions weren't generated for Filament resources, even for super admin.
Fix: Ran php artisan shield:generate --panel=admin --all --no-interaction to create 91 permissions and 8 policies, re-assigned super admin role, and cleared caches.

Ensured MySQL server was running for database operations.
